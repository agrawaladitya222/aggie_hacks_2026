# Aggie Hacks 2026 — Nonprofit Financial Resilience Analytics

End-to-end Python pipeline for IRS Form 990 nonprofit data: **ingestion and feature engineering**, **peer benchmarking**, and a **resilience-at-risk classifier** aligned with the project PRD (`PRD.md`).

## What this repository contains

| Piece | Role |
|--------|------|
| `src/data_pipeline.py` | **Module 1:** Discovers every `*_990.csv` under `data/data_csv/`, concatenates, deduplicates by `(EIN, TaxYear)` (keeping the latest `TaxPeriodEnd`), cleans rows, engineers financial ratios, mission-based **sector**, **peer group** key, **AtRisk** target, and **ResilienceScore** (0–100 rule-based index). |
| `src/peers.py` | **Module 2:** Peer **Z-scores** and **Above/Below peer norm** flags for benchmark metrics within each `PeerGroupID` (sector × size bucket × state). |
| `src/resilience_model.py` | **Module 3:** Trains and selects an **XGBoost** classifier (among logistic regression, random forest, gradient boosting, and XGBoost) to estimate probability of **At Risk** status. Uses **GroupKFold by `TaxYear`**, a **random stratified holdout**, and **temporal split** (train `TaxYear` ≤ 2018, test ≥ 2019) per PRD §11.3. |
| `scripts/build_artifacts.py` | One command to rebuild `data/master_990.csv`, train the model, write `artifacts/`, optional EDA plots under `outputs/`. |
| `tests/` | Pytest coverage for deduplication, feature/target logic, and a smoke train on a subsample. |

## Data layout

- Place annual extracts as CSV files in **`data/data_csv/`** using the pattern **`*_990.csv`** (e.g. `2019_990.csv`, `2020_990.csv`, `2021_990.csv`). Adding new years only requires dropping in another file; the pipeline picks up all matches automatically.
- **`data/master_990.csv`** is generated by the build script: one row per nonprofit–tax year after deduplication, with engineered columns, peer flags, model probabilities, and binary predictions.

## Setup

**1. Create a virtual environment** (recommended so dependencies stay isolated):

```bash
python3 -m venv .venv
```

**2. Activate it**

- **macOS / Linux (bash, zsh):** `source .venv/bin/activate`
- **Windows (cmd):** `.venv\Scripts\activate.bat`
- **Windows (PowerShell):** `.venv\Scripts\Activate.ps1`

Your shell prompt usually shows `(.venv)` when it is active. To leave the environment later, run `deactivate`.

**3. Install dependencies**

```bash
python3 -m pip install -r requirements.txt
```

(After activation, `python` and `pip` typically point at the venv; `python3 -m pip` still works.)

Python **3.10+** recommended.

## Run the pipeline

```bash
# Rebuild master table + train model + write artifacts (default GroupKFold splits: 4)
python3 scripts/build_artifacts.py

# Optional: correlation heatmap and feature-vs-target boxplots (slower)
python3 scripts/build_artifacts.py --eda

# Faster experimentation: limit CV splits or subsample rows used only for *model comparison*
# (the final estimator is always refit on the full dataset when subsampling is used)
python3 scripts/build_artifacts.py --cv-splits 3 --max-train-rows 10000
```

## Model design (high level)

**Target (`AtRisk`, PRD §7.2)** — Binary **1** if any holds: surplus margin below −10%, operating reserves under one month, net-asset decline over 20% YoY, or revenue decline over 25% YoY. **0** otherwise.

**Rule-based `ResilienceScore` (0–100)** — Same PRD weighting on reserves, grant dependence, program expense share, surplus, and debt; used for dashboards and threshold storytelling.

**Classifier inputs** — To avoid **same-period definitional leakage** (the naive case where CY fields that *define* `AtRisk` are also inputs, which inflates AUC toward 1.0), the trained model uses **structural and lag-style signals** instead: revenue mix, expense ratios, debt and asset/liability ratios, expense and contribution **growth**, **prior-year surplus margin** (`SurplusMargin_PY`), **beginning-of-year reserve months** proxy (`ReserveMonths_BOY`), org age, employees, and log revenue/assets. Top importances in the latest run are dominated by **`ReserveMonths_BOY`**, **`ContributionGrowthPct`**, **`AssetLiabilityRatio`**, **`DebtRatio`**, and **`SurplusMargin_PY`**, which matches the intuition that balance-sheet runway and funding momentum precede distress.

**Model selection** — Models are ranked primarily by **temporal ROC-AUC** (then random holdout AUC). **Class imbalance** is handled with `scale_pos_weight` / balanced training as appropriate.

**Decision threshold** — Default 0.5 probability is not always optimal for F1/recall tradeoffs. After selection, a cutoff is tuned on the temporal validation slice by maximizing **Fβ (β = 1.4)** on predicted labels. That value is stored as **`decision_threshold`** in `artifacts/train_metrics.json` and applied when writing **`AtRiskPredicted`** to the master file.
